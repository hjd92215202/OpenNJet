# OpenNJet 在麒麟系统上的国密（NTLS/TLCP）新手使用手册

> 目标读者：第一次接触 OpenNJet + 国密（SM2/SM3/SM4） + 麒麟操作系统的同学。  
> 目标结果：你可以在麒麟上完成 **安装、证书生成、服务端配置、客户端验证、排障与上线前检查**。

---

## 1. 先建立正确认知（新手必看）

在 OpenNJet 里，国密能力通常以 `ntls`（国密双证书握手能力）体现。实操时你会看到：

- 动态 SSL 相关能力支持证书类型 `rsa / ecc / ntls`，其中 `ntls` 对应国密证书。  
- 在健康检查和流量模块里存在 `ntls` 开关逻辑。  

这意味着：**你既可以跑“普通 TLS”，也可以跑“国密 TLS（NTLS/TLCP）”**，具体取决于你的证书与配置。  

---

## 2. 环境准备（麒麟 OS）

> 不同版本麒麟（V10 / 其他发行版）包名略有差异，按系统仓库为准。

### 2.1 基础工具

```bash
sudo yum install -y gcc gcc-c++ make cmake perl wget curl git openssl openssl-devel pcre pcre-devel zlib zlib-devel
```

如你的麒麟使用 `apt`：

```bash
sudo apt update
sudo apt install -y build-essential cmake perl wget curl git libssl-dev libpcre3 libpcre3-dev zlib1g-dev
```

### 2.2 国密工具（建议装）

优先安装系统自带国密工具（例如 `gmssl`），若仓库没有，可用兼容的国密 OpenSSL/GmSSL 方案。

你需要至少具备以下能力：

- 生成 SM2 私钥/证书
- 查看证书算法信息
- 发起国密握手测试

## 3. 快速确认 OpenNJet 是否具备国密能力

### 3.1 查看编译参数与依赖

```bash
./objs/njet -V
```

观察输出中的 SSL 相关信息（是否链接了支持国密/NTLS 的 SSL 实现，是否包含相应模块）。

### 3.2 检查项目中的 `ntls` 能力线索

本仓库中，多个模块存在 `ntls` 证书类型或开关逻辑（例如动态 SSL、健康检查、stream 模块），可作为你判断“该版本支持国密路径”的依据。

---

## 4. 证书准备：国密双证书模型（核心）

国密场景常见是“**签名证书 + 加密证书**”双证书模型。你至少需要：

- CA 证书（可选但推荐）
- 服务端签名证书 + 签名私钥
- 服务端加密证书 + 加密私钥

> 命名建议（避免混乱）：
>
> - `sm2_sign.crt` / `sm2_sign.key`
> - `sm2_enc.crt` / `sm2_enc.key`

### 4.1 一个可参考的目录结构

```text
/etc/njet/certs/
  ├── ca.crt
  ├── sm2_sign.crt
  ├── sm2_sign.key
  ├── sm2_enc.crt
  └── sm2_enc.key
```

### 4.2 基础校验

无论你用 `gmssl` 还是兼容工具，至少做两件事：

1. 证书和私钥匹配
2. 证书算法确认为 SM2/SM3 体系

---

## 5. 最小可用配置思路（先跑通再优化）

> 由于不同构建方式、模块组合、发行包会导致指令名有差异，本节给你“思路 + 验证路径”。

### 5.1 server 侧关键点

1. 开启 SSL 监听
2. 指向国密签名证书/私钥
3. 指向国密加密证书/私钥
4. 在相关模块中开启 `ntls`（若该模块采用布尔开关）
5. 配置你允许的国密套件与协议版本

### 5.2 动态配置/API 关键点

仓库的 swagger 定义中，动态 SSL 证书类型明确包含 `ntls`，并强调国密证书需配置加密相关字段。  
这意味着你用 API 下发证书时，必须保证：

- `type` = `ntls`
- 同时带上签名与加密证书材料（非单证书）

---

## 6. 从 0 到 1：建议实操流程

1. **先本地启动 OpenNJet（普通 TLS）**，确保服务链路正常。  
2. 切换到国密证书（双证书）并启用 `ntls` 路径。  
3. 通过国密客户端工具发起握手测试。  
4. 查看 access/error 日志，确认握手成功、协议协商正确。  
5. 再接入上游、健康检查、动态证书更新能力。

- 普通 OpenSSL 测试：可用于判断是否错误退回了非国密链路。

## 8. 常见问题与定位方法（新手高频）

### 问题 1：配置过了，但握手失败

优先检查：

- 是否真的加载了双证书（签名 + 加密）
- 证书和私钥是否匹配
- 是否启用了 `ntls`
- 协议/套件是否与客户端一致

### 问题 2：动态下发证书失败

优先检查：

- `type` 是否为 `ntls`
- 是否补齐 enc 相关字段
- API JSON 字段拼写、路径权限是否正确

### 问题 3：健康检查一直不健康

优先检查：

- 健康检查的 SSL 配置中是否启用了 `ntls`
- 探测目标服务是否同样支持国密握手

### 问题 4：系统启动成功但业务访问报错

优先检查：

- 防火墙/SELinux/权限（麒麟常见）
- 证书文件权限（建议 `600` 给 key）
- 时间同步（证书有效期）

---

## 9. 生产前建议（非常重要）

1. **灰度**：先小流量国密站点，不要一次全量切换。  
2. **证书轮转**：提前做证书到期提醒和自动替换演练。  
3. **监控**：关注握手失败率、4xx/5xx、上游连接失败。  
4. **回滚预案**：保留上一版配置与证书，确保分钟级回滚。  
5. **基线统一**：同一环境统一 SSL 库版本，避免行为不一致。
